The loan account is the heaviest aggregate in Apache Fineract. The REST API surface that drives it is split into nine cooperating resources: the master `LoansApiResource`, the bulk-loan-officer reassignment endpoint, a point-in-time read API used by external-asset reporting, schedule preview, disbursement-tranche management, guarantors, collateral, rescheduling, and the delinquency taxonomy. Together they expose every operation a credit officer or back-office user can perform on a loan, all under the `/fineract-provider/api/v1` prefix described in the [REST API Overview](/api/overview).

Loan **transactions** (repayments, write-offs, waivers, adjustments) live on their own resource and are documented in [Loan Transactions](/api/loan-transactions).

## `LoansApiResource`

File: `fineract-provider/src/main/java/org/apache/fineract/portfolio/loanaccount/api/LoansApiResource.java`
Class path: `@Path("/v1/loans")`

This is the largest resource in the codebase. It owns the loan lifecycle (submit → approve → disburse → repay → close), application modification, the loan-state dispatcher, and the two newer 2024+ verticals: **approved-amount versioning** and **delinquency actions**.

The state-transition dispatcher is the central piece of logic — every POST against `/v1/loans/{loanId}` looks at `?command=` and routes to a different command handler:

```java
// fineract-provider/.../portfolio/loanaccount/api/LoansApiResource.java
@POST
@Path("{loanId}")
public String stateTransitions(@PathParam("loanId") final Long loanId,
        @QueryParam("command") final String commandParam,
        @Parameter(hidden = true) final String apiRequestBodyAsJson) {
    final CommandWrapperBuilder builder = new CommandWrapperBuilder().withJson(apiRequestBodyAsJson);
    CommandWrapper commandRequest = null;
    if (is(commandParam, "reject"))                commandRequest = builder.rejectLoanApplication(loanId).build();
    else if (is(commandParam, "withdrawnByApplicant")) commandRequest = builder.withdrawLoanApplication(loanId).build();
    else if (is(commandParam, "approve"))          commandRequest = builder.approveLoanApplication(loanId).build();
    else if (is(commandParam, "disburse"))         commandRequest = builder.disburseLoanApplication(loanId).build();
    else if (is(commandParam, "disburseToSavings"))commandRequest = builder.disburseLoanToSavingsApplication(loanId).build();
    else if (is(commandParam, "undoApproval"))     commandRequest = builder.undoLoanApplicationApproval(loanId).build();
    else if (is(commandParam, "undoDisbursal"))    commandRequest = builder.undoLoanApplicationDisbursal(loanId).build();
    else if (is(commandParam, "assignLoanOfficer"))commandRequest = builder.assignLoanOfficer(loanId).build();
    else if (is(commandParam, "unassignLoanOfficer")) commandRequest = builder.unassignLoanOfficer(loanId).build();
    else if (is(commandParam, "recoverGuarantees")) commandRequest = builder.recoverFromGuarantor(loanId).build();
    else if (is(commandParam, "markAsFraud"))      commandRequest = builder.markAsFraud(loanId).build();
    else if (is(commandParam, "forceClose"))       commandRequest = builder.forceCloseLoan(loanId).build();
    // …
}
```

Every command above has a corresponding `*CommandHandler` bean under `fineract-provider/.../portfolio/loanaccount/handler/`. The handlers all delegate to `LoanWritePlatformService` or `LoanApplicationWritePlatformService` for the actual mutations.

### External-id mirror

Every numeric-id POST has an `external-id/{loanExternalId}` twin that resolves the loan via `m_loan.external_id` before dispatching to the same command handler. This is invaluable for integrations that mint their own primary keys upstream.

### Approved-amount versioning

`PUT /v1/loans/{loanId}/approved-amount` records each change in `m_loan_approved_amount_history` so the change history can be reported. The matching `GET` returns the full audit list. This is enforced by `LoanApprovedAmountHistoryRepository` in `fineract-loan`.

### Delinquency actions

The `/{loanId}/delinquency-actions` endpoints let you pause and resume the delinquency clock for one loan — typically for grace periods or pre-litigation holds. The action types are `PAUSE` and `RESUME`. They are persisted to `m_loan_delinquency_action` and consumed by the COB delinquency tasks.

## `LoansPointInTimeApiResource`

File: `fineract-provider/src/main/java/org/apache/fineract/portfolio/loanaccount/api/pointintime/LoansPointInTimeApiResource.java`
Class path: `@Path("/v1/loans/at-date")`

A 2024 addition that reconstructs a loan as it appeared on an arbitrary past date by replaying its journal entries and transactions. Used heavily by `fineract-investor` and external accounting integrations to attest "this is what we held on day X".

| Method | Path | Handler |
| --- | --- | --- |
| GET | `/v1/loans/at-date/{loanId}?date=…` | `retrieveLoanPointInTime` |
| GET | `/v1/loans/at-date/external-id/{loanExternalId}?date=…` | `retrieveLoanPointInTimeByExternalId` |
| POST | `/v1/loans/at-date/search` | `retrieveLoansPointInTime` |
| POST | `/v1/loans/at-date/search/external-id` | `retrieveLoansPointInTimeByExternalIds` |

The POST search forms take a body of `{ "date": "…", "loanIds": [ … ] }` (or `loanExternalIds`) so a single round trip can rebuild many loans at once.

## `LoanScheduleApiResource`

File: `fineract-loan/src/main/java/org/apache/fineract/portfolio/loanaccount/api/LoanScheduleApiResource.java`
Class path: `@Path("/v1/loans/{loanId}/schedule")`

Single endpoint, shared by the **variable installments** add-on. With `?command=calculateLoanSchedule` it returns a preview; without the query param it persists the variable schedule against the loan.

| Method | Path | Handler |
| --- | --- | --- |
| POST | `/v1/loans/{loanId}/schedule` | `calculateLoanScheduleOrSubmitVariableSchedule` |

The handler reads `LoanScheduleType` from the loan product to decide between the standard, variable, progressive, and working-capital schedule generators. See `LoanScheduleGeneratorFactory` in `fineract-loan` for the dispatch.

## `DelinquencyApiResource`

File: `fineract-loan/src/main/java/org/apache/fineract/portfolio/delinquency/api/DelinquencyApiResource.java`

Delinquency configuration is **two-level**: *ranges* are DPD bands (e.g. 1–30, 31–60), and *buckets* are ordered collections of ranges that products bind to. A loan rolls through bucket positions as days-past-due grow; the position drives provisioning, reporting and dunning workflows.

## Permissions

Loan endpoints are gated by fine-grained permissions in `m_permission`. The most important rows:

| Permission | Granted for |
| --- | --- |
| `READ_LOAN` | All `GET /v1/loans*` and `GET /v1/rescheduleloans*`. |
| `CREATE_LOAN` | `POST /v1/loans`. |
| `UPDATE_LOAN` | `PUT /v1/loans/{loanId}` (modify application). |
| `DELETE_LOAN` | `DELETE /v1/loans/{loanId}`. |
| `APPROVE_LOAN` / `UNDO_LOAN` / `APPROVALUNDO_LOAN` | `POST /v1/loans/{loanId}?command=approve|undoApproval`. |
| `DISBURSE_LOAN` / `DISBURSALUNDO_LOAN` | `POST /v1/loans/{loanId}?command=disburse|disburseToSavings|undoDisbursal`. |
| `REJECT_LOAN`, `WITHDRAW_LOAN` | Pre-activation termination. |
| `UPDATELOANOFFICER_LOAN`, `REMOVELOANOFFICER_LOAN`, `BULKREASSIGN_LOAN` | Officer assignment. |
| `FORECLOSURE_LOAN` | `?command=foreclosure`. |
| `MARKASFRAUD_LOAN` | `?command=markAsFraud`. |
| `CREATE_DELINQUENCY_BUCKET`, `UPDATE_DELINQUENCY_BUCKET`, `DELETE_DELINQUENCY_BUCKET` | `DelinquencyApiResource`. |
| `CREATE_RESCHEDULELOAN`, `APPROVE_RESCHEDULELOAN`, `REJECT_RESCHEDULELOAN` | `RescheduleLoansApiResource`. |
| `CREATE_GUARANTOR`, `UPDATE_GUARANTOR`, `DELETE_GUARANTOR` | `GuarantorsApiResource`. |

`ALL_FUNCTIONS` is the superuser bypass; `ALL_FUNCTIONS_READ` covers every `READ_*`. Maker-checker, when enabled globally, adds `CHECKER_<PERMISSION>` rows that approve queued commands.

## Worked example — full submit-to-disbursement flow

```bash
TENANT='Fineract-Platform-TenantId: default'
HDR='Content-Type: application/json'

# 1. Submit
curl -k -u admin:password -H "$TENANT" -H "$HDR" \
  -X POST https://localhost:8443/fineract-provider/api/v1/loans \
  -d '{
    "clientId": 4711, "productId": 7,
    "principal": 1000, "loanTermFrequency": 12,
    "loanTermFrequencyType": 2,
    "numberOfRepayments": 12, "repaymentEvery": 1,
    "repaymentFrequencyType": 2,
    "interestRatePerPeriod": 2.5,
    "interestType": 0, "amortizationType": 1,
    "interestCalculationPeriodType": 1,
    "transactionProcessingStrategyCode": "mifos-standard-strategy",
    "expectedDisbursementDate": "01 April 2024",
    "submittedOnDate": "20 March 2024",
    "dateFormat": "dd MMMM yyyy", "locale": "en"
  }'
# → { "loanId": 911, "resourceId": 911 }

# 2. Approve
curl -k -u admin:password -H "$TENANT" -H "$HDR" \
  -X POST "https://localhost:8443/fineract-provider/api/v1/loans/911?command=approve" \
  -d '{ "approvedOnDate": "25 March 2024", "approvedLoanAmount": 1000,
        "dateFormat": "dd MMMM yyyy", "locale": "en" }'

# 3. Disburse
curl -k -u admin:password -H "$TENANT" -H "$HDR" \
  -X POST "https://localhost:8443/fineract-provider/api/v1/loans/911?command=disburse" \
  -d '{ "actualDisbursementDate": "01 April 2024", "paymentTypeId": 1,
        "transactionAmount": 1000,
        "dateFormat": "dd MMMM yyyy", "locale": "en" }'

# 4. Read back with associations
curl -k -u admin:password -H "$TENANT" \
  'https://localhost:8443/fineract-provider/api/v1/loans/911?associations=all'
```

## Common query parameters on `GET /v1/loans`

| Parameter | Purpose |
| --- | --- |
| `accountNo`, `externalId` | Direct lookup; do not combine with paging. |
| `clientId`, `groupId` | Scope to a borrower. |
| `officeId` | Office-hierarchy scope. |
| `loanOfficerId` | Per-officer portfolio view. |
| `status` | One of `all`, `active`, `pending`, `approved`, `disbursed`, `closedObligationsMet`, `writeoff`, `rescheduled`, `overpaid`. |
| `productId` | Single-product filter. |
| `associations` | Comma-separated: `all`, `repaymentSchedule`, `transactions`, `originalSchedule`, `charges`, `collateral`, `guarantors`, `notes`, `multiDisburseDetails`, `linkedAccount`, `meeting`, `delinquentDetails`. |
| `exclude` | Inverse of `associations`. |
| `fields` | Comma-separated list to project only specific top-level keys. |
| `paged=true`, `offset`, `limit`, `orderBy`, `sortOrder` | Pagination. |
| `sqlSearch` | Legacy free-form predicate (escape carefully; prefer `ClientSearchV2ApiResource` over this). |
