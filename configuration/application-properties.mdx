This page documents every key in the `application.properties` file shipped with Apache Fineract's `fineract-provider` module. The file is the layer‑2 configuration baseline described in the [Configuration Overview](/configuration/overview); each entry uses the `${ENV_VAR:default}` syntax so any value can be overridden at runtime with an environment variable, JVM `-D` flag, or an externally mounted properties file.

**Source files**

- `fineract-provider/src/main/resources/application.properties`
- `fineract-provider/src/main/resources/application-liquibase-only.properties`

The properties below are grouped by subsystem. Within each section the default values are taken verbatim from the file; the environment variable column lists the placeholder name Spring will resolve before falling back to the default.

### Liquibase-only profile

`application-liquibase-only.properties` is loaded when the JVM runs with `--spring.profiles.active=liquibase-only`:

```properties
spring.main.web-application-type=none
```

This switches the Spring Boot app to a non-web type so it runs Liquibase and exits — ideal for CI pipelines and Kubernetes `initContainers`.

## SQL validation patterns

Inline SQL ad-hoc reports run through a regex-based safety filter before execution. The shipped profile blocks blind injection, timing injection, stacked queries, comment-based injection and a handful of backend-detection probes. There are three profiles (`main`, `adhoc`, `column`) which combine the patterns in different orders.

| Pattern key | Purpose |
| --- | --- |
| `fineract.sql-validation.patterns[0]` | `inject-blind` |
| `fineract.sql-validation.patterns[1]` | `detect-entry-point` |
| `fineract.sql-validation.patterns[2]` | `inject-timing` |
| `fineract.sql-validation.patterns[3]` | `detect-backend` |
| `fineract.sql-validation.patterns[4]` | `detect-column` |
| `fineract.sql-validation.patterns[5]` | `detect-out-of-bands` |
| `fineract.sql-validation.patterns[6]` | `inject-stacked-query` |
| `fineract.sql-validation.patterns[7]` | `inject-comment` |
| `fineract.sql-validation.profiles[0]` | `main` (all 8 patterns) |
| `fineract.sql-validation.profiles[1]` | `adhoc` (all 8 patterns) |
| `fineract.sql-validation.profiles[2]` | `column` (omits `detect-column`) |

Disable an individual profile by setting `…profiles[N].enabled=false`. The regex strings themselves are inside `application.properties` (verbatim).

## Logging pattern

The console pattern includes the tenant identifier and the optional correlation id:

```properties
logging.pattern.console=${CONSOLE_LOG_PATTERN:%clr(%d{yyyy-MM-dd HH:mm:ss.SSS}){faint} %clr(${LOG_LEVEL_PATTERN:-%5p}) %clr(${PID:- }){magenta} %clr(%replace([%X{correlationId}]){'\\[\\]', ''}) [%15.15tenantId] %clr(---){faint} %clr([%15.15t]){faint} %clr(%-40.40logger{39}){cyan} %clr(:){faint} %m%n${LOG_EXCEPTION_CONVERSION_WORD:%wEx}}
logging.pattern.level=%5p [${spring.application.name:},%X{traceId:-},%X{spanId:-}]
```
