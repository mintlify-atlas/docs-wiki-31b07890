Apache Fineract is configured at runtime through Spring Boot property resolution: every key inside `application.properties` uses the `${ENV_VAR:default}` placeholder syntax, so setting an environment variable is the canonical way to override configuration without modifying the JAR or mounting a custom properties file. This page lists the `FINERACT_*` names that back the most common settings, plus the additional variables read by Fineract's Docker Compose definitions.

**Source:** `fineract-provider/src/main/resources/application.properties`

## How the mapping works

When Spring Boot sees a property like:

```properties
server.port=${FINERACT_SERVER_PORT:8443}
```

…it resolves the value by, in order:

1. Looking for the system property `FINERACT_SERVER_PORT`.
2. Looking for the environment variable `FINERACT_SERVER_PORT` (Spring also accepts the relaxed-binding form `FINERACT_SERVER_PORT` ↔ `fineract.server-port`).
3. Falling back to `8443`.

That means you can supply any property either by its literal `${...}` name (as listed below) or by Spring Boot's relaxed-binding equivalent of the dotted property key (e.g. `FINERACT_MODE_READ_ENABLED` for `fineract.mode.read-enabled`).

## Docker Compose variables

The Fineract reference Docker Compose stack and the `apache/fineract` container set a small additional layer of environment variables that influence the JVM and the entrypoint scripts:

| Variable | Purpose |
| --- | --- |
| `DRIVERCLASS_NAME` | JDBC driver class loaded by the bootstrap script (e.g. `org.mariadb.jdbc.Driver`). |
| `JAVA_TOOL_OPTIONS` | Extra JVM flags appended to the launch command (heap, GC tuning, debug). |
| `JAVA_OPTS` | Alternative path for extra JVM flags. |
| `TZ` | Container timezone (recommended `UTC`; align with `FINERACT_DEFAULT_TENANTDB_TIMEZONE`). |
| `SPRING_PROFILES_ACTIVE` | Activates Spring profiles, e.g. `liquibase-only` or `test`. |

These three sets work together: the Compose stack exports `FINERACT_*` variables for application properties, the JVM-level variables tune the launcher, and `SPRING_PROFILES_ACTIVE` decides which auxiliary properties file (such as `application-liquibase-only.properties`) is layered on top of the baseline.

## Tips

<Tip>Always rely on environment variables for secrets — never check passwords or API keys into a properties file or container image.</Tip>

<Warning>Setting `FINERACT_INSECURE_HTTP_CLIENT=true` (the default) lets outbound clients accept self-signed certificates. Disable it before going to production.</Warning>

<Tip>Use the `SPRING_PROFILES_ACTIVE=liquibase-only` profile in an init-container to run schema migrations once at deploy time without keeping a Tomcat process alive.</Tip>
